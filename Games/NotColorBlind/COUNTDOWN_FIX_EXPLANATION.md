# 倒计时控制优化说明

## 🎯 需求说明

当玩家选择特殊色块后：
1. ✅ **立即停止当前关卡的倒计时**
2. ✅ 播放成功特效（烟花、连击次数显示）
3. ✅ 切换到下一关时**重置倒计时为初始值**

## ✨ 已实现的修改

### 1. 添加协程引用变量
```csharp
// 倒计时协程引用，用于提前停止倒计时
private Coroutine countdownCoroutine;
```

**作用**: 保存倒计时的协程引用，以便可以在需要时提前停止它。

### 2. 启动倒计时时保存引用
```csharp
// 在 SetupLevel() 方法中
countdownCoroutine = StartCoroutine(StartCountdown());
```

**作用**: 每次开始新关卡时，保存倒计时协程的引用。

### 3. 点击色块时停止倒计时
```csharp
// 在 OnBlockClicked() 方法中
// 停止倒计时
if (countdownCoroutine != null)
{
    StopCoroutine(countdownCoroutine);
    countdownCoroutine = null;
}
```

**作用**: 玩家点击任何色块后（无论对错），立即停止倒计时，防止在播放特效时倒计时还在继续。

### 4. 新关卡自动重置倒计时
```csharp
// 在 ShowResult(true) 中调用 SetupLevel()
SetupLevel(GameData.CurrentLevel);
```

**作用**: `SetupLevel()` 方法会重新启动倒计时协程，实现倒计时重置。

## 🔄 完整流程

### 玩家答对的情况：
1. 玩家点击特殊色块
2. ✅ **倒计时立即停止**
3. 扣除忙币（如果使用了）
4. 增加连续答对次数
5. 播放成功特效（烟花 + 连击文字动画）
6. 等待特效结束（1 秒）
7. ✅ **进入下一关，倒计时重置为 5 秒**

### 玩家答错的情况：
1. 玩家点击普通色块
2. ✅ **倒计时立即停止**
3. 重置连续答对次数
4. 切换到 ResultScene

## 📋 关键改进点

### 改进前的问题：
- ❌ 点击色块后倒计时继续运行
- ❌ 可能在播放特效时倒计时结束，导致逻辑冲突
- ❌ 进入下一关后倒计时可能不是从 5 秒开始

### 改进后的优势：
- ✅ 点击色块后立即停止倒计时，逻辑清晰
- ✅ 特效播放期间不会有倒计时干扰
- ✅ 每关开始都是全新的 5 秒倒计时
- ✅ 避免协程泄漏（及时停止和重新创建）

## 💡 技术要点

### 协程管理最佳实践：
1. **保存引用**: 使用 `Coroutine` 类型变量保存协程引用
2. **及时停止**: 在不再需要时使用 `StopCoroutine()` 停止
3. **清空引用**: 停止后将引用设为 `null`，避免重复停止
4. **重新创建**: 新关卡开始时启动新的协程实例

### 为什么这样做：
- Unity 的协程一旦启动就会独立运行
- 如果不手动停止，即使场景切换也可能继续执行
- 保存引用可以精确控制协程的生命周期
- 避免多个倒计时协程同时运行导致的 bug
